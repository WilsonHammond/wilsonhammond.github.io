# Remediation Roadmap

## Phase 1, Containment and Access Revocation

Objective: Stop further exposure and remove dormant vendor access.

Actions:

* Force credential reset for all third party accounts
* Revoke inactive Application Programming Interface tokens
* Disable dormant vendor integrations
* Increase monitoring for unusual data access
* Preserve logs for legal and forensic review

## Phase 2, Vendor Governance Rebuild

Objective: restore ownership, inventory accuracy, and review discipline.

Actions:

* Rebuild the full vendor inventory
* Assign a business owner and security owner to every critical vendor
* Require updated security documentation
* Reassess vendors that handle Protected Health Information or payment data
* Create a vendor review calendar with executive visibility

## Phase 3, Identity and Access Management Modernization

Objective: reduce excessive third party privileges and restore least privilege.

Actions:

* Implement quarterly third party access recertification
* Require multifactor authentication for vendor access
* Remove shared accounts
* Limit Application Programming Interface permissions by business need
* Review privileged roles monthly

## Phase 4, Governance Tooling and Evidence Management

Objective: centralize risk evidence and reduce manual tracking failure.

Actions:

* Implement Governance, Risk, and Compliance workflow tracking
* Standardize vendor evidence requests
* Add automated expiration alerts
* Track exception approvals with due dates
* Connect control evidence to risk register records

## Phase 5, Board Level Recovery Reporting

Objective: restore executive confidence through measurable progress.

Actions:

* Report open high risks monthly
* Track residual risk reduction
* Report human control metrics
* Track vendor review completion and evidence rejection rates
* Present cost, progress, and control maturity trends to leadership
