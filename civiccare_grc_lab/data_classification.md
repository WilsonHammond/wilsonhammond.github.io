# Data Classification Model

## Restricted Data

Restricted data creates the highest legal, regulatory, contractual, or mission impact if mishandled.

Examples

1. Electronic protected health information  
2. Consumer financial information  
3. Cardholder data  
4. Controlled Unclassified Information  
5. Sensitive personally identifiable information  
6. Authentication secrets  
7. Privileged administrator logs  
8. Encryption keys

Expected Handling

1. Access must be limited by role and business need.  
2. Multifactor authentication must protect privileged and remote access.  
3. Encryption must protect data at rest and in transit.  
4. Logging must capture access, configuration changes, and administrative actions.  
5. Data sharing must be governed by contract, data use agreement, or approved business purpose.  
6. Evidence must be retained for audit and customer review.

## Confidential Data

Confidential data would damage the company, customer trust, or operations if exposed.

Examples

1. Internal risk assessments  
2. Vendor security reviews  
3. Incident response plans  
4. Network diagrams  
5. Security policies  
6. Vulnerability reports

Expected Handling

1. Internal access only.  
2. Version control and approval required.  
3. No public sharing.  
4. Retention must follow policy.

## Internal Data

Internal data supports normal business operations.

Examples

1. General procedures  
2. Training calendars  
3. Internal project notes  
4. Non sensitive reports

Expected Handling

1. Available to employees with a business reason.  
2. Not intended for public release.

## Public Data

Public data is approved for release.

Examples

1. Public website copy  
2. Approved marketing material  
3. Public privacy notices  
4. Published job postings

Expected Handling

1. Must be approved before publication.  
2. Must not include regulated or confidential data.
